Command Types 
MS-DOS shell provides 3 categories of commands :


  #  Internal Commands


  #  External Commands 


  #  Batch Files or Command Files


                         Internal Commands
 
The internal commands are directly executed from the RAM. These


commands are part of the COMMAND.COM  file which is loaded into


the RAM at the time of booting.


For example, VER, DIR, REN, DEL, COPY etc.



All these internal commands are discussed later in the CBT.



                         External commands
           


The external commands are the names of the programs stored in the


disk files. Before these programs can be  executed  they  must be


loaded from the disk into the RAM.


For example, CHKDSK, FORMAT, BACKUP etc.



All these external commands are also discussed later in the CBT.

                   Batch commands
 
Batch  files  are  text  files that contain a list of internal,


external or batch commands which are executed in  sequence when


when the batch file is  executed ( by typing in the name of the


file). AUTOEXEC.BAT is  a  batch  file  which  gets  executed 


automatically  on  booting. This file contains certain start-up 


instructions. All the batch files will have an extension .BAT.

                         Redirection


      



Redirection refers to the  ability of a program or a MS-DOS command




to  choose an alternative device for input or output. Most programs




and commands have a default device. For example, when you enter the




DIR command, the computer assumes that the directory listing should




be  displayed on  the screen. This is because  the  default  output




device is the console (CON).




The redirection symbols used by MS-DOS are >, <, >>.


                       



The default output device can be overridden by using the redirect-




ion  symbol. The output redirection symbol  (>) can be used to re-




direct the output either to a printer or to a disk file.




DIR > PRN  will  result  in  sending the directory listing to  the




printer instead of the default output device ie. the console.




DIR > C:\DIR.vin  will  similarly  result in storing the directory




listing in the file.




The redirection symbol for appending to a file is the >> symbol.




DIR A: >> DIR.vin will append the output to the existing DIR.vin




file.


type this command    c:\tree /f /a > files.txt 


this will give a tree structure of all files and folder inside c:\ drive in to 


a ascii (/a) file files.txt



MS-DOS can receive input from text file instead of accepting data




from keyboard. To indicate that inputs are coming from a file and




not from the keyboard the input  redirection  symbol (<) is used.





                         Pipe Character



Piping allows you  to redirect the output  of one command or program




as input to another command or program. Several interlinked commands




can thus  be  combined to form a single  complex command.  The  Pipe




symbol  can  be  generated pressing  the vertical bar symbol  |. A




series of commands or programs that are inter-dependent and generate




intermediate output for one  another can  be  combined into a single




command  by using a combination  of redirection and pipe characters.
